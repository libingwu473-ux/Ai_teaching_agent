# 项目使用手册

## 1. 文档说明

本文档面向本项目的管理员、教师和学生用户，说明系统当前已实现功能、典型操作流程和使用注意事项。

项目名称：基于 Dify 的高职线上自主学习教育智能体  
技术形态：`Django 4.2 + DRF + JWT + Vue 3 + Vite + Element Plus`  
当前角色体系：`admin`、`teacher`、`student`

## 2. 系统概览

系统围绕“学生与 Dify 智能体对话学习，教师查看过程并评分，管理员维护教师与平台配置”展开，主要模块如下：

- 学生端：聊天学习、查看历史会话、上传文件
- 教师端：专业管理、班级管理、学生管理、CSV 批量导入、评分复核、统计看板、评分参数配置
- 管理员端：教师管理、Dify 平台配置、后台总览

## 3. 角色与权限

### 3.1 管理员

管理员可以执行以下操作：

- 登录系统后台
- 新增、编辑、停用教师账号
- 重置教师密码
- 配置 Dify 平台连接参数
- 查看管理员控制台

管理员不能直接维护学生数据录入流程，学生由教师负责管理。

### 3.2 教师

教师可以执行以下操作：

- 维护专业信息
- 创建和维护自己名下的班级
- 录入学生、修改学生信息、停用学生
- 批量导入班级学生 CSV
- 重置学生密码
- 查看学生会话、聊天记录和评分结果
- 手动触发评分、批量重算评分
- 配置评分权重和阶段消息门槛

教师只能管理自己名下班级的数据。

### 3.3 学生

学生可以执行以下操作：

- 登录系统
- 进入聊天页与 Dify 智能体对话
- 查看自己的历史会话
- 在聊天中上传文件
- 修改自己的登录密码

## 4. 登录与首次使用

### 4.1 登录入口

统一访问入口：

- `https://ktxm.chat.43d.cn/login`

### 4.2 登录信息

登录接口为：

- `POST /api/auth/login/`

登录成功后系统返回：

- 用户基础信息
- `role`
- `must_change_password`
- JWT `access_token`
- JWT `refresh_token`

### 4.3 首次修改密码

系统创建的管理员、教师、学生账号通常会带有 `must_change_password=true` 标记。首次登录后应立即修改密码。

修改密码接口：

- `POST /api/auth/change-password/`

## 5. 管理员使用说明

### 5.1 教师管理

管理员进入教师管理页面后可完成：

- 新增教师
- 编辑教师姓名、邮箱、启用状态
- 停用教师账号
- 重置教师密码

说明：

- 新建教师时，系统会自动设置角色为 `teacher`
- 重置密码后，教师会再次被要求首次登录修改密码

### 5.2 Dify 配置

管理员可在 Dify 配置页面维护以下参数：

- API Base URL
- API Key
- App ID
- Chatflow ID
- 是否校验 SSL
- 请求超时
- 最大重试次数

这些参数保存在数据库单例表 `DifyConfig` 中，修改后立即生效，不依赖重启后端。

## 6. 教师使用说明

### 6.1 专业管理

教师可在专业管理页面进行：

- 新增专业
- 编辑专业名称和描述
- 停用专业

约束：

- 专业编码唯一
- 若专业下仍有启用中的班级，则不能停用该专业

### 6.2 班级管理

教师可在班级管理页面：

- 查看自己的班级列表
- 新建班级
- 修改班级名称
- 停用班级

约束：

- 班级属于当前教师
- 同一专业下班级名称唯一
- 班级下仍有启用学生时不能停用

### 6.3 学生管理

教师在某个班级下可进行：

- 查看学生列表
- 单个新增学生
- 编辑学生姓名、性别、所属班级
- 停用学生
- 重置学生密码

规则：

- 学号对应 `username`
- 若新增时密码留空，则默认密码为学号
- 默认要求学生首次登录修改密码
- 密码长度至少 6 位

### 6.4 CSV 批量导入学生

支持通过 CSV 批量导入某个班级学生。

CSV 表头要求：

```text
学号,密码,专业,班别,姓名,性别
```

导入规则：

- 使用严格事务导入
- 任意一行校验失败，则整批不导入
- 密码为空时默认等于学号
- 性别支持：`男`、`女`、`其他`、空
- CSV 中专业和班别应与当前导入目标班级一致

模板下载接口：

- `GET /api/teacher/students-csv-template/`

导入接口：

- `POST /api/teacher/classes/{class_id}/class-students/import/`

### 6.5 统计看板

教师可查看以下统计数据：

- 学生总数
- 活跃学生数
- 会话总数
- 平均得分
- 阶段完成率
- 最近 7 天日活

阶段完成率采用“按会话封顶的平均进度口径”，与阶段最少消息数配置联动。

### 6.6 评分与复核

教师可：

- 查看某学生所有评分记录
- 查看单次评分详情
- 修改教师评分、评语、状态
- 手动触发单个会话评分
- 触发全量重算

相关接口：

- `POST /api/teacher/scoring/trigger/`
- `POST /api/teacher/scoring/recalculate-all/`

### 6.7 评分参数设置

教师可配置：

- 阶段完成度权重
- 流程遵循度权重
- 时间投入权重
- 参与度权重
- 最小会话分钟数
- 满分值

同时还可配置工作流阶段的 `expected_min_messages`，用于：

- 判断某阶段是否完成
- 计算教师看板的阶段完成率

## 7. 学生使用说明

### 7.1 进入聊天

学生登录后进入聊天页面：

- `/chat`

系统支持：

- 新建会话
- 查看历史会话
- 继续历史会话
- 删除会话

### 7.2 发送消息

聊天消息通过后端代理发送到 Dify：

- 前端调用 `POST /api/chat/send/`
- 后端以 `text/event-stream` 流式返回

学生在聊天过程中可实时看到模型输出内容。

### 7.3 文件上传

学生可上传文件到 Dify：

- `POST /api/chat/upload/`

前提：

- 当前学生账号必须已绑定 `DifyUserMapping`

### 7.4 历史会话

支持：

- 会话列表查询
- 会话详情查询
- 会话消息查询
- 删除会话

相关接口：

- `GET /api/chat/conversations/`
- `GET /api/chat/conversations/{id}/`
- `GET /api/chat/conversations/{id}/messages/`
- `DELETE /api/chat/conversations/{id}/delete/`

## 8. 系统评分逻辑

当前实现的评分模型为四维评分：

- 阶段完成度
- 流程遵循度
- 时间投入
- 参与度

评分结果写入：

- `StudentScore`
- `ScoreDetail`

说明：

- 阶段判定依赖 `WorkflowStage.expected_min_messages`
- 如果 Dify 工作流未正确输出阶段信息，评分仍可生成，但阶段相关维度可能偏低

## 9. 典型使用流程

### 9.1 管理员初始化

1. 管理员登录
2. 配置 Dify 平台参数
3. 创建教师账号

### 9.2 教师开班

1. 教师登录
2. 创建专业
3. 创建班级
4. 单个录入学生或导入 CSV
5. 通知学生登录

### 9.3 学生学习

1. 学生登录
2. 修改初始密码
3. 进入聊天页学习
4. 与智能体进行多轮对话

### 9.4 教师复核

1. 教师查看学生学习记录
2. 触发评分或重算评分
3. 查看评分详情
4. 填写教师评分和评语

## 10. 注意事项

- 本项目开发环境数据库为 `SQLite`，并发写入能力有限
- 流式聊天依赖 SSE，生产环境建议使用 ASGI 服务
- Dify 平台配置改动由管理员统一维护
- 学生聊天行为只有通过本系统前端或同步服务接入时，才能被本地完整纳入分析
- 若 Dify 工作流阶段变量未配置完善，阶段分析与评分将受影响

## 11. 相关文件与模块

- 后端配置：`backend/config/settings.py`
- 用户与班级模型：`backend/apps/users/models.py`
- Dify 配置与工作流模型：`backend/apps/dify_integration/models.py`
- 评分模型：`backend/apps/scoring/models.py`
- 前端路由：`frontend/src/router/index.js`
