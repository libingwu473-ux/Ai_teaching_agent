# 项目安装手册

## 1. 文档目的

本文档用于指导在当前项目基础上完成本地开发环境部署、初始化和启动验证。

项目目录结构：

- `backend/`：Django 后端
- `frontend/`：Vue 3 前端

## 2. 环境要求

### 2.1 基础环境

- Python `3.11` 或兼容版本
- Node.js `18+`
- npm

### 2.2 后端依赖

`backend/requirements.txt` 当前内容对应以下主要依赖：

- `django>=4.2,<5.0`
- `djangorestframework>=3.14`
- `djangorestframework-simplejwt>=5.3`
- `django-cors-headers>=4.3`
- `django-filter>=23.5`
- `httpx>=0.25`
- `python-decouple>=3.8`
- `whitenoise>=6.6`

### 2.3 前端依赖

`frontend/package.json` 当前主要依赖：

- `vue`
- `vue-router`
- `pinia`
- `axios`
- `element-plus`
- `@element-plus/icons-vue`
- `@microsoft/fetch-event-source`
- `vite`

## 3. 获取项目

将项目代码放置于本地目录，例如：

```powershell
cd d:\
```

当前项目根目录为：

```text
d:\test004
```

## 4. 后端安装步骤

### 4.1 创建并激活虚拟环境

如未创建虚拟环境，可执行：

```powershell
cd d:\test004\backend
python -m venv ..\.venv
```

激活虚拟环境：

```powershell
cd d:\test004
.\.venv\Scripts\Activate.ps1
```

### 4.2 安装依赖

```powershell
cd d:\test004\backend
pip install -r requirements.txt
```

### 4.3 配置环境变量

后端使用 `python-decouple` 读取环境变量，关键配置包括：

```env
SECRET_KEY=django-insecure-dify-edu-secret-key-change-in-production
DEBUG=True
DIFY_API_BASE_URL=https://dify.chat.43d.cn/v1
DIFY_API_KEY=your-api-key
DIFY_APP_ID=
DIFY_CHATFLOW_ID=
DIFY_VERIFY_SSL=False
```

说明：

- 上述配置仅作为数据库单例配置的初始化默认值
- 系统实际运行优先读取数据库中的 `DifyConfig`
- 如无 `.env` 文件，也可先使用默认值启动

### 4.4 执行数据库迁移

```powershell
cd d:\test004\backend
python manage.py migrate
```

默认数据库为：

- `backend/db.sqlite3`

### 4.5 初始化管理员账号

```powershell
python manage.py seed_admin
```

默认账号：

- 用户名：`admin`
- 密码：`admin123`

如需重置管理员密码：

```powershell
python manage.py seed_admin --reset-password
```

### 4.6 初始化默认教学工作流

```powershell
python manage.py seed_workflow
```

该命令会：

- 创建默认教学流程
- 创建 3 个默认阶段
- 若系统中没有教师账号，会生成 `default_teacher`

### 4.7 启动后端

```powershell
python manage.py runserver 8000
```

## 7. Dify 对接配置说明

系统通过数据库单例 `DifyConfig` 读取配置，字段包括：

- `api_base_url`
- `api_key`
- `app_id`
- `chatflow_id`
- `verify_ssl`
- `timeout`
- `max_retries`

管理员在前端配置后可立即生效，无需重启服务。

## 8. 评分系统初始化说明

评分相关配置来自 `ScoringConfig` 单例，默认项包括：

- `stage_completion_weight`
- `sequence_adherence_weight`
- `time_investment_weight`
- `engagement_weight`
- `min_session_minutes`
- `max_score`

首次读取时，系统会根据 `settings.py` 中的 `SCORING_CONFIG` 自动创建默认配置。

## 10. 运行验证

完成安装后，至少验证以下内容：

### 10.1 后端验证

- 访问登录接口可返回结果
- 数据库迁移成功
- `seed_admin` 命令成功执行

### 10.2 前端验证

- 登录页可打开
- 登录后能根据角色进入对应页面
- 学生聊天页可以建立 SSE 连接

## 11. 生产部署注意事项

- 当前默认数据库为 SQLite，仅适合开发或小规模场景
- 生产环境建议切换到 PostgreSQL
- 流式聊天依赖 SSE，建议使用 `Uvicorn` 或 `Daphne` 运行 Django ASGI
- `DEBUG` 应设置为 `False`
- `SECRET_KEY` 需替换为正式值
- `CORS_ALLOWED_ORIGINS` 应按正式域名收敛配置

## 12. 故障排查

### 12.1 前端无法登录

检查项：

- 后端是否已启动在 `8000`
- 浏览器中是否存在过期 token
- 用户是否被停用

### 12.2 聊天无法返回流式内容

检查项：

- Dify 配置是否正确
- 当前学生是否存在 `DifyUserMapping`
- 网络是否能访问 Dify API
- 生产环境是否错误使用了不支持流式的 WSGI 方案

### 12.3 学生导入失败

检查项：

- CSV 表头是否正确
- 学号是否重复
- 密码长度是否不少于 6 位
- 专业、班别字段是否与目标班级一致
